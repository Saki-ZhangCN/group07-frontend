# 《在线教育智慧教学平台》接口文档

> 实现基线：2026-07-01。接口统一使用 `/api` 前缀，响应结构为 `{code,message,data}`。直播接口仅保留前端定义，不属于本次实现与验收范围。认证、用户、课程、作业、学情、数据看板和运营管理接口已在主后端注册并接入数据库。学员数据按JWT中的用户身份隔离；管理接口要求管理员角色；作业发布、批改及班级学情接口要求讲师或管理员角色。

## 教师分类题库组卷

`GET /homework/question-bank` 要求教师权限。`courseId` 必填，`difficulty`、`type`、`keyword`、`page`、`pageSize` 可选；接口校验课程属于当前教师，并只返回该课程 `category` 下的习题。`POST /homework` 接收 `courseId`、`title`、`deadline`、`totalScore`、`passScore`、`status`、`exerciseIds`，服务端再次校验任课关系和全部习题分类。

## 课程章节与资源

- `GET /course-content/teacher/{courseId}`：教师查看本人课程章节及全部审核状态资源。
- `POST /course-content/teacher/{courseId}/sections`：创建课程章节（父级）。
- `POST /course-content/teacher/sections/{sectionId}/lessons`：在章节内创建小节。
- `DELETE /course-content/teacher/sections/{sectionId}`：删除章节及其小节。
- `DELETE /course-content/teacher/chapters/{chapterId}`：删除本人课程小节。
- `POST /course-content/teacher/chapters/{chapterId}/upload?kind=video|material`：以 `multipart/form-data` 上传MP4或PDF，上传后状态为 `pending`。
- `GET /course-content/admin/pending`：管理员查询待审核教学资源。
- `POST /course-content/admin/{kind}/{id}/review`：管理员通过或拒绝资源。
- `GET /course-content/student/{courseId}`：已选课学生获取章节，仅返回审核通过的视频和PDF。

## 一、接口规范说明

### 1、基础URL

- 开发环境：`http://localhost:5173/api`（前端开发服务器，Vite代理转发至后端8080端口）
- 测试环境：`http://test.example.com/api`
- 生产环境：`https://api.example.com/api`

> **说明**：前端开发服务器运行在 `http://localhost:5173`，通过Vite代理配置将 `/api` 请求转发至后端Spring Boot服务（默认端口8080）。后端服务地址为 `http://localhost:8080/api`。

### 2、请求头

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| Authorization | String | 是 | Bearer Token，格式：`Bearer {token}` |
| Content-Type | String | 是 | application/json;charset=utf-8 |

### 3、响应格式

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {}
}
```

### 4、状态码说明

| 状态码 | 说明 |
|--------|------|
| 200 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未授权，Token无效或已过期 |
| 403 | 禁止访问，权限不足 |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |
